# Phase 4: Verify

### 4. Verify

After completing all steps, run a single verification pass using tool output as objective evidence.

**4a — Run toolchain.**

Run the full toolchain in one command:

```
pnpm lint && pnpm typecheck && pnpm test && pnpm knip && pnpm lint:clones
```

**Confirm:** Zero errors + warnings. Test count not dropped. Each Tests table name in output. No new knip findings. Zero code clones.

Runs ONCE. On failure, fix and re-run the full chain.

**Known issue — knip false positives in `.git-worktrees/` paths.** If `pnpm knip` fails in the worktree but all other checks pass, run this fallback from the **main workspace root**:

1. `git merge --no-commit --no-ff <branch> && pnpm knip; git merge --abort`
2. Knip exits 0 on main → known false positive (record and proceed). Exits non-zero → real failure (fix it).

**4c — Confirm clean and track first-pass quality.**

**Gate Function:** For every claim: (1) run the tool command, (2) read the actual output, (3) verify it matches success criteria, (4) then claim the result. No memory-based claims.

After §4b, every dimension must be clean. Unchecked dimensions = skipped — go back and check. Architectural issues that cannot be fixed mechanically → **Blocked diagnostic**.

**First-pass tracking:** Record whether each dimension was clean on first check or needed a fix. Report in §5a (e.g. "21/21 first-pass clean" or "19/21, fixed 2: readonly array in X, block comment in Y"). Exclude inapplicable conditional dimensions (16-18, 21) from denominator. Dims 19-20 are informational — always in denominator but don't block for pre-existing issues. Quick-check catches in §3 count as first-pass clean.

Once all dimensions are confirmed clean, proceed to §5.
