# Task 077: EnvExampleRedactor

> **Status:** Pending
> **Phase:** L (Transformers & Guard)
> **Layer:** pipeline
> **Depends on:** (none — Phase L transformers are independent)

## Goal

Implement a ContentTransformer that redacts values in `.env.example`, `.env.sample`, and `.env.template` files — replacing `KEY=value` with `KEY=***` line by line — so the model sees variable names (useful context) while values are stripped, reducing tokens by ~30–50%.

## Architecture Notes

- Implements existing ContentTransformer interface. No new interface.
- ADR-010: use branded types (RelativePath, FileExtension, InclusionTier) from core/types.
- Pipeline transformer: no constructor params, no external package; pure string/line logic. Non-format-specific (`fileExtensions = []`) so path-based detection runs in `transform`.
- Path detection: basename (last path segment) starts with `.env` AND ends with `.example`, `.sample`, or `.template`. Covers `.env.example`, `.env.sample`, `.env.template`, `.env.local.example`, `.env.production.sample`, etc. Does NOT match `.env` (handled by ContextGuard never-include), `.env.local`, or `.env.development`.
- Output: line-by-line redaction. Comment lines (`# ...`) kept for documentation context. Blank lines kept. `KEY=value` lines (with optional `export ` prefix, with or without quotes around value) have the value replaced with `***`. Lines without `=` kept unchanged.
- Wiring: Insert EnvExampleRedactor after autoGeneratedSkipper, before htmlToMarkdownTransformer in the transformers array.

## Steps

### Step 1: Implement EnvExampleRedactor

Create `shared/src/pipeline/env-example-redactor.ts`.

- Imports: `ContentTransformer` from `#core/interfaces/content-transformer.interface.js`; `FileExtension`, `RelativePath` from `#core/types/paths.js`; `InclusionTier` from `#core/types/enums.js`.
- Module constant: `EMPTY_EXTENSIONS: readonly FileExtension[] = []`.
- Module constant: `ENV_EXAMPLE_SUFFIXES: readonly string[] = [".example", ".sample", ".template"]`.
- Module-level regex: `VALUE_LINE_RE = /^(export\s+)?([A-Za-z_][A-Za-z0-9_]*)=(.*)$/` — matches optional `export ` prefix, a variable name, `=`, and the rest as value.
- Helper function `isEnvExamplePath(path: RelativePath): boolean`:
  - Extract basename: `const segments = path.split("/"); const basename = segments.length > 0 ? (segments[segments.length - 1] ?? path) : path;`
  - Convert basename to lowercase for case-insensitive suffix matching.
  - Return true if the lowercase basename starts with `.env` AND ends with one of the suffixes in `ENV_EXAMPLE_SUFFIXES`. Use `ENV_EXAMPLE_SUFFIXES.some(suffix => lower.endsWith(suffix))`.
  - Return false otherwise.
- Helper function `redactLine(line: string): string`:
  - If the line is empty or starts with `#` (comment), return line unchanged.
  - Match against `VALUE_LINE_RE`. If no match, return line unchanged.
  - Extract groups: `prefix` (group 1, the `export ` or undefined), `key` (group 2), `_value` (group 3, unused).
  - Return `${prefix ?? ""}${key}=***`.
- Class: `EnvExampleRedactor` with `readonly id = "env-example-redactor"`, `readonly fileExtensions: readonly FileExtension[] = EMPTY_EXTENSIONS`.
- In `transform(content, _tier, filePath)`: if `content.length === 0` return content. If `!isEnvExamplePath(filePath)` return content. Otherwise: split content by `"\n"`, map each line through `redactLine`, join with `"\n"`, return the result. Explicit return type `string`.
